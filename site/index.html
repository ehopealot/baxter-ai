Self-hosted · Open source · Discord

The agent that lives in your server.

Baxter is a Discord bot you run on your own box. Every message it's addressed in spawns one scoped agent run that can reply, browse the web, execute code in an offline sandbox, keep notes, and act on a schedule — driven by whatever tool-calling model you point it at.

The fleet

One command brings up the whole thing.

baxter up starts the default services under docker compose, each with a restart policy. Two more are opt-in. Memory, keys, and schedule live on a named volume, so restarts and rebuilds don't cost the agent anything it learned.

  • running

    discord

    Holds the gateway websocket. Answers DMs, @mentions, replies, and channel messages — and wakes when someone reacts to one of its own posts.

  • running

    heartbeat

    Ticks every minute and fires due tasks, so the agent can act unprompted. One-shot or cron, delivered back to Discord or email.

  • running

    codapi

    The code sandbox. Each execution is an ephemeral container with no network, no capabilities, and a read-only filesystem.

  • opt-in

    mail

    Polls a dedicated AgentMail inbox and replies in-thread. One API key — no Google account, no OAuth screen, nothing to renew.

  • opt-in

    voice

    Joins a voice channel, listens for a request, and DMs the speaker the result of what it dispatched.

  • on demand

    shell

    baxter shell is your own console: chat to the same agent, or drive its tools directly with /slash commands. Works over SSH too.

Abilities

Tools it actually has, not tools it can describe.

Each run is handed a small set of purpose-built command-line tools. There is no bare shell: a tool exists or it doesn't, and the allow-list is the boundary.

Browses the real web

A keyless fetcher for plain pages, a Chromium driver for interactive ones, and a persistent profile so logins carry across conversations.

Gets past bot walls

A second, anti-detect browser on patched Firefox for sites that fingerprint automation — masked at the engine level, not with a spoofed user agent.

Writes and runs code

Python or Node in the offline sandbox, with numpy, pandas, and BeautifulSoup already there. Charts, images, and PDFs come back as Discord attachments.

Remembers

A shared memory file, per-channel notes, and one file per project — all on the volume, all readable from every surface it lives on.

Keeps a schedule

Ask for a daily digest or a one-off reminder and it books the task itself, in your timezone. The heartbeat fires it whether or not anyone is talking.

Writes its own skills

When it works out how to use something new, it saves the procedure as a skill and loads it on the next run. Learned skills can never shadow built-in ones.

Prefers good sources

A curated gateway to structured APIs — live scores, geocoding — so it reaches a real source instead of scraping, and never handles the key itself.

Moderates, where you allow it

It can delete other people's messages only in channels where you granted Manage Messages. Everywhere else Discord itself refuses.

The brain

Swap the model without touching anything else.

The same prompts, skills, and tools run on any of three drivers. Switch with one command, restart, and everything it has learned comes with it.

Default

OpenRouter

Any tool-calling model, pay per token. No Anthropic account required. Runs well and cheaply on mid-tier models.

baxter harness openrouter openai/gpt-4o

Alternative

Claude Code

Drive it with the Claude CLI instead. Log in once — the credential persists on the config volume.

baxter harness claude

Alternative

Local model

Any OpenAI-compatible endpoint: Ollama, LM Studio, llama.cpp, vLLM. Nothing leaves the machine. Needs tool calling.

baxter harness local qwen3
Guardrails

The limits are code, not prompt text.

Anything reading the agent's inbox or channel is writing part of its input, so the rules that matter are the ones a model can't talk its way around. These are enforced by the daemon, the container, or Discord itself.

Sender allow-list
Fails closed on the mail surface — an empty list processes no mail at all.
Self-gating
The agent never acts on its own messages, so it can't talk itself into a loop.
Run budget
A per-channel hourly cap ends a bot-to-bot ping-pong that rate limits alone wouldn't.
Send caps
Daily ceilings on both email and Discord, counted at the actual send.
Offline execution
Sandboxed code runs with network: none, so a compromised run has nowhere to send anything.
Credential boundary
Tokens are stripped from every run's environment and written to a 0600 file that, on the default driver, no read primitive can reach.
Confined reads
On the default driver every read and write resolves inside the working directory and refuses to escape.
Host-locked requests
The data gateway pins each source's origin, so a key can only ever be sent to that source.
No membership commands
Nothing exists to kick, ban, or assign roles — alongside the permissions you don't grant.
Schedule ceilings
A minimum recurrence, a task limit, and a daily fire cap bound what a booked task can become.
Setup

Running in four steps.

You need Docker with compose v2, make, an OpenRouter key, and a Discord application you control.

  1. 01

    Configure

    Copy the example environment file and fill in your keys. Every variable is commented in place.

    cp app/.env.example app/.env
  2. 02

    Pick the brain

    OpenRouter is the default. Name any model that supports tool calling — the driver is already set in the example file.

    # app/.env
OPENROUTER_API_KEY=sk-or-...
OPENROUTER_MODEL=openai/gpt-4o
  3. 03

    Connect Discord

    Create an application, enable the Message Content intent, paste the bot token into app/.env, and invite the bot to your server.

  4. 04

    Bring it up

    Install the CLI once, then start the fleet. It's live in your server as soon as the gateway connects.

    ./install.sh && baxter up

Then, day to day

  • baxter statusWhat's running.
  • baxter logs discordFollow one service.
  • baxter shellTalk to it directly, or drive its tools.
  • baxter backupArchive its entire state — memory, skills, schedule.
  • baxter restoreRoll it back, or clone it onto another box.
  • baxter updatePull, rebuild, restart, in one shot.

Give your server a resident.
